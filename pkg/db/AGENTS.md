# pkg/db Agent Notes

This package contains the JSON-backed catch data store, data model, migrations,
and generated read-only view/clone helpers.

## Local Rules

- Treat `Data` as persisted state. Schema changes usually require a
  `CurrentDataVersion` bump, migrator, backup behavior tests, and compatibility
  tests for old on-disk data.
- `db_view.go` and `db_clone.go` are generated by the `go:generate` commands in
  this package. Regenerate them instead of hand-editing generated accessors.
- Store methods should preserve clone/view boundaries so callers cannot mutate
  cached state accidentally.
- Keep migration errors explicit; silent data loss is worse than a loud failure.

## Tests

- Run `go test ./pkg/db -count=1` after data model, migration, view, clone, or
  persistence changes.
- Run `go test ./pkg/db ./pkg/catch ./pkg/dnet -count=1` when data changes feed
  catch state or Docker network behavior.
