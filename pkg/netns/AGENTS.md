# pkg/netns Agent Notes

This package installs yeet network namespace services, renders firewall state,
and embeds the `netns-scripts/` shell helpers.

## Local Rules

- Treat firewall, namespace, and systemd operations as privileged side effects.
  Prefer unit tests around rendered config, command selection, and generated
  unit/env files before live testing.
- Keep `netns-scripts/` compatible with the env files generated by Go code.
- Support both nft and iptables backends unless intentionally dropping a path.
- Run live validation when behavior depends on real namespaces, firewall tools,
  DHCP clients, or systemd.

## Tests

- Run `go test ./pkg/netns -count=1` after namespace or firewall changes.
- Run `go test ./pkg/netns ./pkg/catch ./pkg/svc -count=1` when service setup
  consumes changed netns behavior.
